# Theory

## What is Server Hardening?
Server hardening is the process of securing a server by reducing vulnerabilities, disabling unnecessary services, applying updates, and configuring security controls.

The main goal is to reduce the attack surface and improve the overall security of the system.

## Why Linux Server Hardening is Important
Linux servers are widely used in professional environments to host services, applications, websites, databases, and monitoring tools.

A poorly configured server can be exposed to:
- Unauthorized access
- Brute-force attacks
- Malware
- Data leaks
- Service disruption

## Main Hardening Areas

### 1. System Updates
Keeping the system updated helps fix security vulnerabilities and improve stability.

### 2. User Management
User accounts should be controlled carefully. Each user should have only the required permissions.

### 3. SSH Security
SSH is commonly used for remote administration. It should be secured by changing default settings and limiting access.

### 4. Firewall Configuration
A firewall helps control incoming and outgoing network traffic.

### 5. File Permissions
Correct file permissions protect sensitive system files and user data.

### 6. Service Management
Unnecessary services should be disabled to reduce security risks.
